# WorkspaceGraph Product Philosophy

Version: 0.1 (Draft)

------------------------------------------------------------------------

# Introduction

Dokumen ini menjelaskan filosofi di balik setiap keputusan desain
WorkspaceGraph.

WorkspaceGraph tidak dibangun sebagai aplikasi AI, aplikasi catatan,
atau aplikasi graph.

WorkspaceGraph dibangun sebagai **Knowledge Operating System**.

------------------------------------------------------------------------

# The Problem

Saat ini pengetahuan pengguna tersebar di berbagai tempat:

-   Folder
-   Dokumen
-   Catatan
-   Chat AI
-   Email
-   Template
-   SOP
-   Proyek

Akibatnya:

-   sulit dicari
-   tidak saling terhubung
-   mudah terlupakan
-   AI tidak memahami konteks secara utuh.

WorkspaceGraph hadir untuk menyatukan semuanya.

------------------------------------------------------------------------

# Knowledge Over Files

WorkspaceGraph memandang bahwa pengguna sebenarnya tidak bekerja dengan
"file".

Pengguna bekerja dengan:

-   ide
-   hubungan
-   keputusan
-   pengalaman
-   proses

File hanyalah media penyimpanan.

------------------------------------------------------------------------

# AI Must Read, Not Remember

AI tidak boleh menjadi tempat penyimpanan pengetahuan.

Sebelum bekerja AI harus:

1.  Menemukan informasi yang relevan.
2.  Membaca konteks.
3.  Menyusun pemahaman.
4.  Menghasilkan solusi.
5.  Menulis hasil kembali ke Workspace.

Dengan demikian seluruh pengetahuan tetap berada di Vault.

------------------------------------------------------------------------

# Knowledge Grows Like a Graph

Setiap catatan baru harus dapat dihubungkan dengan catatan lain.

Nilai sebuah catatan meningkat ketika ia memiliki hubungan yang bermakna
dengan catatan lain.

Graph bukan sekadar visualisasi, tetapi representasi struktur
pengetahuan.

------------------------------------------------------------------------

# Workspace Before Conversation

Percakapan AI hanyalah salah satu aktivitas.

Workspace adalah pusatnya.

Chat harus mampu:

-   membaca Workspace,
-   membuat perubahan,
-   memperbarui tugas,
-   membuat dokumentasi,
-   memperbarui Graph.

------------------------------------------------------------------------

# Long-Term Thinking

WorkspaceGraph dirancang agar Workspace tetap berguna selama
bertahun-tahun.

Pergantian model AI, framework, maupun teknologi tidak boleh membuat
pengetahuan pengguna hilang.

------------------------------------------------------------------------

# Open By Design

Workspace harus:

-   mudah dipindahkan,
-   mudah dicadangkan,
-   mudah dibuka dengan editor lain,
-   mudah dikelola menggunakan Git.

Tidak boleh ada ketergantungan terhadap format tertutup.

------------------------------------------------------------------------

# Extensible System

WorkspaceGraph dirancang sebagai platform.

Kemampuan baru ditambahkan melalui plugin, bukan dengan mengubah inti
sistem.

Dengan cara ini aplikasi dapat berkembang tanpa kehilangan stabilitas.

------------------------------------------------------------------------

# Human + AI Collaboration

AI bukan pengganti manusia.

AI adalah kolaborator.

Manusia menetapkan tujuan.

Workspace menyediakan konteks.

AI membantu melakukan pekerjaan.

Hasil akhirnya kembali menjadi pengetahuan milik manusia.

------------------------------------------------------------------------

# Product Identity

WorkspaceGraph adalah:

-   AI Workspace
-   Knowledge Operating System
-   Markdown-first Platform
-   Graph-native Workspace
-   Plugin-ready Platform
-   AI-independent System

WorkspaceGraph bukan sekadar aplikasi pencatat dengan AI.

Ia adalah sistem kerja yang menghubungkan pengetahuan, proyek, tugas,
dan AI dalam satu Workspace yang utuh.

------------------------------------------------------------------------

# Guiding Question

Setiap fitur baru harus menjawab pertanyaan berikut:

1.  Apakah fitur ini memperkuat Workspace?
2.  Apakah fitur ini menjaga Markdown sebagai Source of Truth?
3.  Apakah fitur ini membuat Graph lebih bermakna?
4.  Apakah AI tetap hanya sebagai pekerja?
5.  Apakah data tetap dimiliki pengguna?
6.  Apakah fitur ini masih berguna jika model AI diganti?

Jika salah satu jawabannya adalah "tidak", maka fitur tersebut harus
ditinjau ulang.

------------------------------------------------------------------------

# Closing

WorkspaceGraph tidak mengejar menjadi AI paling pintar.

WorkspaceGraph mengejar menjadi tempat terbaik bagi manusia untuk
membangun, menghubungkan, dan mengembangkan pengetahuan sepanjang
hidupnya.
